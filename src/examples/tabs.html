Tabs Component

Dashbase uses the semantic <tab-list> / <tab-panel> pattern with a tiny behavior shim for selection, roving focus, and panel visibility. Arrow keys, Home, and End all work once the behavior file is loaded.

Default Tabs

Overview

Dashbase prefers semantic HTML for structure, CSS for presentation, and minimal vanilla JavaScript only when the platform has a real behavior gap.

Activity

Recent activity, release status, and follow-up work all fit well inside tab panels when the information is mutually exclusive.

Members

Use tabs when users need to switch between peer sections without leaving the current context.

Underline Variant

Use the underline variant when the tab list should feel lighter and closer to text navigation than a segmented control.

Invoices remain in the same page flow, but only one panel is exposed at a time.
