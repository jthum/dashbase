Carousel Component

Dashbase uses <carousel> when a sequence of slides should remain horizontally scrollable but still gain previous/next controls, dot navigation, and keyboard support. The foundation stays native with scroll snapping; the behavior shim only coordinates state.

Feature Tour

Each slide fills the viewport width by default, which makes this a good fit for media previews, onboarding tours, and marketing storytelling.

Launch report

Track signups and activation in the same view

Combine acquisition, trial conversion, and workspace activation in a single release dashboard without switching tools.
Segment comparison

Compare cohorts without losing the narrative

Pin campaign slices side by side and keep the commentary directly attached to the chart or table you are reviewing.
Approvals

Move from draft to approval without leaving the workspace

Drafts, comments, and final approvals all live inside the same surface, so the carousel can walk teams through a multi-step flow.

Looping Card Rail

Set a smaller --carousel-slide-size when you want a card rail that peeks ahead to the next item. Add data-loop to wrap around the ends.
