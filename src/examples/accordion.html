Accordion Component

Dashbase styles the native <details> / <summary> disclosure pattern directly, keeping the built-in semantics and interaction model intact.

Basic Accordions

How does Dashbase approach interaction?

Dashbase prefers native HTML and CSS first, then adds the smallest possible amount of vanilla JavaScript only when the platform has a real behavior gap.

Does this require any JavaScript?

No. The accordion itself is the browser's native disclosure widget. Dashbase is only styling <details> and <summary>.

Can the content be richer than plain text?

Yes. Content can be text, lists, form fields, or action rows. The native disclosure behavior still works the same way.

Deployment checklist

Use accordions for optional detail, not for hiding critical content users must discover immediately.

  • Verify examples against component contracts
  • Check light and dark mode
  • Confirm keyboard and focus behavior
Support policy

Dashbase targets modern evergreen browsers and chooses graceful degradation over heavy compatibility layers.
