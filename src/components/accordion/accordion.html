Accordion / Disclosure Component

Dashbase styles the native <details> / <summary> pattern as a disclosure first. Plain disclosures work well for FAQ-style content without extra wrappers. For richer content and a more card-like accordion shell, use the optional <accordion-panel> wrapper. Density can be tuned with .small and .large.

Plain Disclosures

How does Dashbase approach interaction?

Dashbase prefers native HTML and CSS first, then adds the smallest possible amount of vanilla JavaScript only when the platform has a real behavior gap.

Does this require any JavaScript?

No. The accordion itself is the browser's native disclosure widget. Dashbase is only styling <details> and <summary>.

Can the content be richer than plain text?

Yes. Content can be text, lists, form fields, or action rows. The native disclosure behavior still works the same way.

Density Variants

Compact disclosure

Use .small when space is tight, such as FAQ lists or dense settings pages.

Large disclosure

Use .large when the trigger should feel more spacious or match a more editorial layout.
